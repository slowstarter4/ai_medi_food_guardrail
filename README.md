Medication & Food Safety Guardrail System

프로젝트 개요
이 프로젝트는 처방전 및 식품·영양 성분표를 분석하여
약물–음식(또는 건강기능식품) 간 잠재적인 상호작용 위험을 사전에 탐지하고
사용자에게 예방 중심의 경고를 제공하는 AI 기반 가드레일 시스템을 구축하는 것을 목표로 한다.

병원 외 일상 환경에서 발생하는 복약·식단 충돌 사고는
정보가 없어서가 아니라, 정보가 분산되어 있고 해석이 어렵기 때문에 발생한다.
본 시스템은 해당 정보를 연결·구조화하여
섭취 직전에 위험 여부를 간단히 확인할 수 있도록 돕는다.

이 시스템은 의료 진단이나 처방을 제공하지 않는다.

문제 정의
사용자는 병원에서 처방받은 약물과
일상적으로 섭취하는 음식, 음료, 건강기능식품 간의 충돌 위험을
직접 판단하기 어렵다.

처방전, 약 봉투, 식품 성분표에 필요한 정보는 모두 존재하지만
전문 용어 위주로 분산 제공되어 직관적인 비교가 어렵고
사고는 대부분 섭취 이후에 인지된다.

본 프로젝트는 사고 발생 이후 대응이 아닌
사전 예방 중심의 가드레일 시스템을 제안한다.

프로젝트 범위 (MVP)
포함
- 특정 약물 카테고리 1종
- 금기·주의 사례 약 20~30개
- 이미지 기반 입력(OCR)
- 룰 기반 위험 판단

제외
- 질병 진단
- 복약 또는 식단 추천
- 의료적 의사결정

Input
- 이미지
  - 처방전
  - 약 봉투
  - 식품·건강기능식품 성분표
- 텍스트(선택)
  - 사용자의 현재 상태
  - 기저 질환 정보
- 공공 데이터
  - 의약품 정보 및 약물–성분 상호작용 데이터

Output
- 위험 수준 표시 (high / caution / none)
- 위험 요약 설명
- 위험 발생 근거 및 출처
- 사용자 경고 메시지

Output JSON Interface (Draft)
이 JSON은 시스템의 최종 판단 결과 형식에 대한 초안이며
구현을 위한 고정 스펙이 아닌 팀 합의용 인터페이스이다.

```json
{
  "input": {
    "drug": ["string"],
    "intake": ["string"]
  },
  "result": {
    "risk_level": "high | caution | none",
    "risk_summary": "string"
  },
  "reason": [
    {
      "type": "drug-food | drug-supplement",
      "description": "string",
      "evidence": "string"
    }
  ],
  "source": ["e약은요"],
  "disclaimer": "의료적 판단이 아닙니다"
}
```

```json
[
  {
    "drug": "와파린",
    "intake": "시금치",
    "risk_level": "high",
    "description": "와파린 복용 시 비타민 K가 많은 시금치 섭취는 항응고 효과를 감소시킬 수 있습니다.",
    "evidence": "비타민 K는 혈액 응고를 촉진함"
  }
]
```

기술 구성 (초안)
- OCR: 처방전 및 성분표 이미지 텍스트 추출
- NLP: 약물명 및 성분명 구조화
- Rule-based Logic: 공공 의료 데이터 기반 위험 판단
- 간단한 UI 또는 콘솔 기반 결과 출력

팀 역할
- NLP: 텍스트 구조화 및 파이프라인 설계
- Computer Vision: OCR 처리
- Data Analysis: 위험 케이스 정의 및 룰 정리
- Planning: 사용자 시나리오 및 메시지 설계
